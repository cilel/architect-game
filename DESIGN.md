## 🎯 游戏概念与目标

### 核心目标
让用户通过**"模块化拼装"**体验从拿地、规划到内部空间布置的建筑师全流程。

### 游戏定位
介于**"模拟经营"**（如《模拟城市》）与**"创作工具"**（如《The Sims》或《Minecraft》）之间。

### 目标受众
- 🏠 建筑爱好者
- 🎓 建筑/设计相关专业学生
- 🎮 休闲模拟游戏玩家

---

## 🎮 核心玩法与机制

### 1. 网格与体块系统 (Grid & Volume)
- 采用**基于网格的坐标系统**，降低对齐难度
- 核心机制是**"体块堆叠"**，让非专业用户也能快速勾勒出建筑轮廓
- 支持不同高度的楼层叠加

### 2. 资源与预算限制
- 引入**预算系统**，增加游戏挑战性
- **住宅**：强调"舒适度/坪效"
- **公共建筑**：强调"功能流线/容纳人数"

### 3. 环境模拟 (Environmental Simulation)
- ☀️ **光照模拟**：实时日照角度、阴影计算
- 🌳 **植被影响**：绿化对建筑评分的影响
- 🌤️ **气候变化**：不同天气条件下的视觉效果

### 4. 评分与反馈系统
游戏会根据以下维度给出即时反馈：
- ✅ **采光**：窗户面积、朝向、遮挡情况
- ✅ **结构合理性**：承重墙分布、跨度合理性
- ✅ **美观度**：比例平衡、色彩搭配
- ✅ **功能性**：空间利用率、动线设计

## 🏢 建筑类型与组件系统

### 建筑类型

| 类型 | 设计重点 | 典型案例 |
|------|---------|---------|
| **住宅** | 动静分区、采光、隐私 | 公寓、别墅、联排住宅 |
| **公共建筑** | 人流疏散、大跨度空间、标志性造型 | 图书馆、博物馆、剧院 |
| **商业建筑** | 展示面、人流引导、品牌识别 | 商场、办公楼、餐厅 |

### 组件分类

#### 结构类
- 梁 (Beam)
- 柱 (Column)
- 楼板 (Floor Slab)
- 承重墙 (Load-bearing Wall)

#### 围护类
- 玻璃幕墙 (Glass Curtain Wall)
- 窗户 (Window) - 多种尺寸
- 入户门 (Entrance Door)
- 屋顶 (Roof) - 平顶/坡顶

#### 环境类
- 树木 (Tree)
- 长椅 (Bench)
- 人行道 (Sidewalk)
- 水池/喷泉 (Water Feature)
- 路灯 (Street Lamp)

## 🖱️ 用户交互设计 (UX)

### 操作模式

| 模式 | 视角 | 用途 |
|------|------|------|
| **建造模式** | Top-down / 等距视角 | 平面布局、结构搭建 |
| **漫游模式** | 第一人称视角 | 进入建筑内部感受尺度 |

### 快捷键设计

| 按键 | 功能 |
|------|------|
| `Tab` | 切换构建目录 |
| `R` | 旋转组件 |
| `Delete` | 删除选中组件 |
| `Ctrl + Z` | 撤销上一步 |
| `Ctrl + Y` | 重做 |
| `Space` | 切换建造/漫游模式 |
| `F12` | 拍照模式（隐藏UI导出截图） |

### 核心交互特性
- 🔲 **吸附功能 (Snapping)** - 网页端操作的灵魂
- 🎯 **智能对齐** - 自动对齐到网格或相邻组件
- 📐 **尺寸显示** - 实时显示距离和角度

## 🛠️ 技术架构建议

### 推荐技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **3D 引擎** | Three.js / Babylon.js | Three.js轻量灵活，Babylon.js功能更全 |
| **前端框架** | React / Vue 3 | 处理UI菜单、库存系统，与3D引擎桥接 |
| **物理引擎** | Cannon.js / Rapier.js | 结构碰撞检测、重力塌陷模拟 |
| **后端** | Node.js + Supabase | 保存用户模型、截图分享、排行榜 |
| **部署** | Vercel / Netlify | 快速部署、自动CI/CD |

- 🚀 **Instanced Mesh** - 实例化渲染处理大量重复组件（如玻璃窗）
